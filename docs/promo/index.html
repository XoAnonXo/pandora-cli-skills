Audience split

One product surface, three ways in.

Fastest path Agents

Pandora should feel instantly usable inside Codex, Claude Code, and any MCP-capable runtime.

  • Start read-only with bootstrap, capabilities, schema, policy list, and profile list.
  • Use local stdio MCP for the default self-custody execution path.
  • Offer remote HTTP MCP as a read-only planning gateway when teams need shared discovery.
npm install && npx pandora mcp
Deterministic path CLI

Operators need the site to show that Pandora is not only agent-friendly. It is also direct, scriptable, and auditable.

  • Use the human setup sequence from the repo: init-env, doctor, build, help.
  • Keep safe exploration visible with bootstrap, capabilities, schema, and policy/profile listing.
  • Highlight quote, scan, trade, LP, claim, mirror, sports, and operations receipts as real commands.
npm install && npm run init-env && npm run doctor && npm run build && npx pandora help
Embed path SDK

Hardcore devs should immediately see that the same surface can be embedded from TypeScript or Python.

  • TypeScript package: @thisispandora/agent-sdk@alpha with local stdio or remote HTTP MCP clients.
  • Python package: pandora-agent with the same bootstrap-oriented flow.
  • Generated contracts, CJS/ESM support, and package-local inspection are strong builder proof.
npm install @thisispandora/agent-sdk@alpha

Command-level proof

Marketing should still feel executable.

The strongest Pandora story is not abstract. It is concrete: bootstrap safely, expose tools through MCP, stay deterministic in the CLI, and embed the same contract surface through TypeScript or Python.

Safe bootstrap

Canonical read-only entry point for agents and humans before secrets or mutation.

npm install && npx pandora --output json bootstrap

Local stdio MCP

Default self-custody path when the agent runs on the same machine as Pandora.

npm install && npx pandora mcp

Remote planning gateway

Hosted read-only HTTP MCP path for discovery, schema, recipes, audit, and receipts.

npm install && npx pandora mcp http --auth-scopes capabilities:read,contracts:read,help:read,schema:read,operations:read,scan:read,quote:read,portfolio:read,mirror:read,sports:read,network:indexer,network:rpc,network:polymarket,network:sports

CLI discovery and quoting

Promote real command-level proof, not vague feature labels.

npx pandora scan --output json --limit 10
npx pandora quote --output json --market-address 0x... --side yes --amount-usdc 25

TypeScript SDK

Public alpha package for apps that want the same contract surface from code.

npm install @thisispandora/agent-sdk@alpha
const client = createPandoraAgentClient({ command: 'pandora', args: ['mcp'] })

Python SDK

Mirror the same story for Python teams that want local or remote Pandora clients.

pip install pandora-agent==0.1.0a15
