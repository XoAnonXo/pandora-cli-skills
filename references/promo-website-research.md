# Promotional Website Research

Date: 2026-03-13

## Goal

Define the strongest patterns to borrow for a Pandora promotional website aimed at:

- agent users who want a fast path into Codex, Claude, and MCP
- operators who want deterministic CLI workflows
- builders who want SDK surfaces they can embed

## External references reviewed

### Pandora

Source: https://thisispandora.ai

Strengths to borrow:

- high-conviction brand energy instead of a generic docs landing page
- market-native motion and visual tension
- strong sense that Pandora is an active protocol, not a passive library

### Model Context Protocol

Source: https://modelcontextprotocol.io

Strengths to borrow:

- protocol-first framing
- ecosystem explanation before implementation detail
- strong mental model for how tools connect to agents

### Stripe

Source: https://stripe.com

Strengths to borrow:

- immediate trust and product clarity above the fold
- strong command/code posture without burying the marketing story
- crisp audience transition from overview to implementation

### Supabase

Source: https://supabase.com

Strengths to borrow:

- self-serve builder ergonomics
- obvious product paths for different user intents
- strong pairing of high-level pitch with practical quickstarts

### Vercel AI

Source: https://vercel.com/ai

Strengths to borrow:

- template and starter framing for fast activation
- builder-first positioning for modern AI workflows
- clear linkage between product value and implementation velocity

### OpenAI Codex

Source: https://openai.com/codex/

Strengths to borrow:

- outcome-oriented language for agentic work
- emphasis on delegation, parallelism, terminal execution, and PR flow
- proof that the product helps advanced developers ship faster

### Anthropic Claude Code

Source: https://claude.com/product/claude-code

Strengths to borrow:

- codebase-aware agent framing
- strong connection to real developer workflows
- credible positioning for agent-plus-terminal collaboration

## Implications for Pandora

The promotional surface should:

1. Lead with immediate actions for agent users, not a passive headline.
2. Split the story into `Agents`, `CLI`, and `SDK` quickly.
3. Show command-level proof early with real Pandora entrypoints.
4. Treat `Skills` as leverage multipliers, not a footnote.
5. Present multiple art directions so the team can choose between launch energy and durable docs credibility.

## Instant CTA note

I did not find an official deep-link standard for launching prefilled tasks directly into Codex or Claude Code from a public marketing page. The implementation therefore uses:

- an official destination for each product
- copy-to-clipboard prompts or commands so the action is still immediate

This keeps the UX honest while preserving the "open in Codex / Claude" intent.
