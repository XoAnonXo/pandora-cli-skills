# Add Brier Tracking and model score brier Command

Introduce forecast logging and calibration scoring to measure whether probabilities generated by market/consensus/BYOM/PF are actually predictive.

## Problem Statement

Without calibration tracking, we cannot quantify forecast quality or edge. Current CLI emits probabilities but does not score them after resolution.

## Findings

- Article identifies Brier score as the standard calibration metric and operational necessity.
- Current `watch` command already centralizes repeated snapshots and is ideal for optional forecast logging hooks.
- CLI already has historical and resolve surfaces that can be joined for scoring windows.

## Proposed Solutions

### Option 1: Forecast ledger + new scoring command (recommended)

**Approach:** Log forecasts to JSONL with provenance and build `model score brier` aggregations.

**Pros:**
- Explicit audit trail
- Works with existing commands
- Supports rolling windows and source-level diagnostics

**Cons:**
- Requires careful join logic for resolved outcomes

**Effort:** 6-8 hours

**Risk:** Medium

---

### Option 2: Inline scoring only in watch output

**Approach:** Compute score opportunistically when outcomes are available.

**Pros:**
- Smaller implementation

**Cons:**
- No persistent dataset
- Poor historical analysis support

**Effort:** 3-4 hours

**Risk:** Medium

## Recommended Action

Implement `--track-brier` in `watch` and add command:
`pandora model score brier [--source <name>] [--window-days <n>] [--group-by source|market|competition]`.
